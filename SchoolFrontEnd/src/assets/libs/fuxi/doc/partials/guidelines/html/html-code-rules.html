Regras de codificação

Esta secção apresenta as regras de codificação que a equipa segue para a declaração de markup e estilos.

Nunca usamos estilos inline

Toda a estilização é feita através de classes CSS. Não misturamos o markup com o estilo.

Recomendado
Não recomendado

Definimos corretamente a estrutura do documento

Para manter o código HTML semanticamente correto e de acordo com os standards definidos respeitamos a estrutura do documento.

Recomendado
Não recomendado

Usamos alt nas imagens

Em todas as imagens colocamos o atributo alt, ainda que a vazio, de forma a que os screen readers consigam providenciar contexto a imagens.

Recomendado
Não recomendado

Referências para ficheiros css externos no head da página

Colocamos todos os ficheiros externos de CSS dentro do elemento head do documento.

Recomendado

Não usamos código JavaScript inline

Não colocamos código JavaScript diretamente nos elementos HTML.

Recomendado
Não recomendado

Tags em lowercase

Escrevemos sempre as nossas tags em lowercase.

Recomendado
Não recomendado

Usamos as tags h1 a h6 para definição dos títulos

Por questões de SEO e de semântica, sempre que aplicável, usamos as tags de heading para os títulos.

Recomendado
Não recomendado

Navegação dentro da estrutura de lista

Por razões semânticas, a navegação deve ser feita com recurso a lista não ordenada ul.

Recomendado
Não recomendado

Uso desnecessário de divs para construção da estrutura

De maneira a manter o markup limpo e reduzir o número de linhas de código, evitamos o uso de elementos supérfluos.

Recomendado
Não recomendado
