Data attributes do wizard

Para que um wizard funcione corretamente é necessário definir uma série de data-attributes.

Datta-attribute Descrição
data-wizard Declarado no elemento que encapsula todo o wizard. Instancia o objeto como wizard.
data-wiz-mode Declarado no elemento que encapsula todo o wizard. Define o modo do wizard. Aceita os valores INSERT | EDIT | VIEW.
data-validate-on-previous Declarado no elemento que encapsula todo o wizard. Define se a validação deve ser disparada quando se retrocede no wizard. Aceita os valores true | false.
data-wiz-prev Declarado no elemento que encapsula todo o wizard. Deve referir o botão de retroceder no wizard e aceita ID ou CLASS.
data-wiz-next Declarado no elemento que encapsula todo o wizard. Deve referir o botão de avançar no wizard e aceita ID ou CLASS.
data-wiz-complete Declarado no elemento que encapsula todo o wizard. Deve referir o botão de finalizar wizard e aceita ID ou CLASS.
data-step-opened Declarado no elemento que encapsula todo o wizard. Refere-se ao step aberto por defeito. Apenas aceita inteiros a partir de 1.
data-client-validator Declarado no elemento que encapsula todo o wizard ou ao nível do step. Função de validação no cliente. Aceita o nome da função que trata da validação. A definição desta função fica a cargo de cada equipa de desenvolvimento. Caso este data-attribute não se encontre definido ao nível do step, será utilizada a função definida no mesmo atributo que se encontra definido no elemento que encapsula o wizard.
